# Workflow Checklist

Use this checklist for a full course-sharing PPT page image workflow.

## Intake

- Topic and exact title
- Presenter name/date if needed
- Audience and course context
- Target duration
- Required page-image count or likely page-image count
- Whether the final PPTX wrapper is standalone or appended to an existing PPT
- Source files and links
- Reference PPT or style examples
- Must-use keywords, concepts, cases, laws, or textbook terms

## Source Handling

- Extract `.docx`/`.txt`/`.csv`/`.md` content as UTF-8.
- On Windows PowerShell 5.x, avoid Bash-style `&&`; run command steps separately.
- Browse official/primary sources for unstable facts.
- Keep source notes for:
  - laws and effective dates
  - official statistics
  - regulator notices
  - public enforcement or inspection cases
  - textbook-only points

## Reference PPT Audit

- Page count and section rhythm
- Cover and closing style
- Directory style
- Header/page marker pattern
- Dominant colors
- Font family
- Table and matrix grammar
- Icon style
- Information density
- Image background style

## Content Plan

- Build a thesis and arc before image generation.
- Use claim titles rather than topic labels when possible.
- Give every page image a role:
  - cover
  - agenda
  - why it matters
  - definition
  - history/timeline
  - framework/process
  - key mechanism
  - case
  - comparison
  - trend/career
  - summary
  - closing
- Add a formal closing page when the PPT image set is for a report.

## Iteration Log

Track user changes such as:

- add textbook points
- add audience-specific application points
- change closing wording
- remove Q&A
- update cases
- regenerate specific pages only

Update the outline file before regenerating images.
